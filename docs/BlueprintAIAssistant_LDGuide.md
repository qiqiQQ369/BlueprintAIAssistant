# Blueprint AI 助手 · 地编使用手册

> 面向：关卡/地编 / 初级蓝图使用者。
> 目的：用最少操作，让 AI 帮你把常见蓝图脚手架 10 秒搭好；出错了能一键反馈。

---

## 1. 第一次使用（3 分钟设置）

### 1.1 打开面板

主菜单 `Window → Blueprint AI Assistant`（或工具栏里找到对应按钮），弹出浮动/停靠窗口。

### 1.2 确认模型已连通（只需一次）

窗口顶部会显示一行：`Provider=xxx | Model=xxx | Endpoint=xxx`。

1. 点 **「连接测试」** 按钮。
2. 下方输出区显示 `连接成功。模型已响应：OK` 即代表连通，可以继续。
3. 如果失败：
  - `Edit → Editor Preferences → Plugins → Blueprint AI Assistant Settings` 里，检查当前 Provider 的 API Key / Endpoint 是否填写，找开发同学要配置。
  - 常见问题：公司网络代理、API Key 过期、余额用完。

### 1.3 打开一个蓝图再操作

**重要**：AI 助手需要你**先打开一个蓝图资源**（BP_Something），它才能拿到上下文并把节点生成到对应的 EventGraph / Function 图里。

### 1.4 多轮追问（Phase 6.A）

- 在输入框里**连续追问**时，插件会把**最近若干轮**「你说过什么 / 助手答过什么」注入到发给模型的内容里（默认 **6 轮**），并尽量附带**当前面板里已有 DSL 步骤摘要**和**最近一次失败摘要**，方便你说「把上一步改成…」而不必全文重贴。
- 需要开新话题时，点 **「清空多轮历史」**（不会删除已生成的 DSL 预览，只清对话记忆与失败摘要）。
- 可在 **项目设置 → Blueprint AI Assistant → 对话 | Phase 6.A** 中关闭或调整轮数、长度上限。

### 1.5 模糊需求怎么办？（分诊 / 澄清 / 计划）

当你的需求比较模糊、或看起来会涉及多个蓝图（例如：背包/任务/UI 联动），插件会尽量**先帮你把需求拆清楚**，而不是强行生成一份“看似能执行但实际不对”的 DSL。

- **澄清模式**：若缺少关键细节（例如触发事件、目标蓝图/对象来源、变量/Widget 名等），面板会自动给出 2～4 个问题。你在问题下方填写答案后，点 **「基于补充信息生成 DSL」** 即可继续。
- **计划模式（跨蓝图）**：若检测到可能是跨蓝图/系统级需求，面板会先生成一个实施计划，并列出若干“可落到单蓝图 DSL 的步骤”。你可以逐条点 **「生成该步 DSL」**，并在执行前先打开对应蓝图（按步骤里的提示即可）。

## 2. 最快上手：点一下"场景快捷按钮"

面板顶部有一排预设场景按钮（按键开门 / 触发器 / 延时 Spawn / 蒙太奇 / 受伤掉血 / PrintString / 前方 LineTrace）。

**操作**（以「按键交互开门」为例）：

1. 打开你想加逻辑的蓝图（比如 `BP_MyCharacter`）。
2. 点 **「按键交互开门」** 按钮。
3. 面板会自动：
  - 填入一段预设 prompt
  - 调用模型生成 DSL 步骤

补充：执行 DSL 后，插件会对**本次生成/涉及的节点做一次轻量自动排版**（只移动节点，不改变连接），让执行线尽量从左到右、分支往下展开，减少"线很乱"的情况。**v2 增强**：列宽/行高按节点**真实尺寸**自适应（若编辑器已打开，取 Slate 测量结果；否则经验值），**Branch 的 True/False、Sequence 的 Then0/Then1… 会按分支在源节点上的垂直顺序**自上而下排；新节点若挡住了既有节点，会把**被挡的既有节点整体下移避让**，不再压在一起。若该蓝图**已在 Blueprint 编辑器中打开**，排版后还会尽量调用编辑器自带的**拉直连接线**（与 "Straighten Connections" 一致）；编辑器未打开时跳过此步，不影响规则排版。

**v3 增强（与路线图 §4.3 一致，已交付）**：长执行线可自动插 **Knot** 中继；纯数据节点可**卫星贴靠**到主要消费它的执行节点附近；**Comment** 框可随本次节点包络调整，并对用户手调范围采用**只扩不缩**；复杂图下对执行线**分簇**再 Straighten，比全选一次更稳。对应开关在 `DSL | 自动排版`：`bLayoutInsertKnotsForLongWires`、`bLayoutAttachDataNodesAsSatellites`、`bLayoutAutoResizeCommentBoxes`（均受总开关 `bAutoLayoutAfterExecute` 约束）。

> 不想让插件动节点？项目设置 → `Blueprint AI Assistant` → `DSL | 自动排版` 里把 `bAutoLayoutAfterExecute` 关掉即可；`bLayoutAvoidCollisions` 控制是否做碰撞避让（v2）；v3 三个开关见上段。

> 可靠性相关设置（Phase 5）：如果模型输出偶尔不是严格 JSON，可以打开 `DSL | 可靠性` 下的 `bAutoRetryOnceOnDslParseFailure`，插件会在「生成 DSL」解析失败时自动回灌错误并重试 1 次。若批量/单步执行中途失败，可打开 `bAutoRetryOnceOnDslExecFailure`，插件会先 Undo 回滚本次执行，再自动修复并重试 1 次。若你希望“尽量做完能做的步骤，再统一看哪些失败”，可打开 `bAggregateDslExecutionFailures`。如果你希望解析更严格、避免“猜测/纠错”，可关闭 `bEnableDslSchemaAutoSanitize`（但更容易触发解析失败与自动重试）。

- 下方「DSL 步骤预览」列出每一步

1. 查看步骤无误后：
  - **推荐**：从第 1 步开始，逐行点行末的 **「执行」** 按钮，看蓝图里每步的变化
  - **熟练后**：勾选想执行的步骤，点 **「执行所选 DSL 步骤」** 批量跑完
2. 回到蓝图编辑器观察节点是否已经就位，点击 `Compile` 编译看能否通过。

> Tips：
>
> - 按钮上悬停可以看到该场景的完整 prompt 文案。
> - 生成不理想？直接在输入框里修改 prompt 再点「生成可执行步骤（DSL）」，会**覆盖**之前的 DSL 预览。

## 3. 自己写需求：输入自然语言

如果内置场景不完全贴合你的需求，可以直接在输入框里用中文写你要做的事，越具体越好。例如：

```
我想做一个按键交互开门蓝图：
- 按下 E 键触发；
- 向前方做 300 长度 LineTrace（Visibility 通道）；
- 命中 BP_Door_01 就调用其 Open 方法，否则打印 "No Door"。
```

然后点：

- **「生成建议」**：只给文字说明，不动蓝图（用于先了解原理）
- **「生成步骤清单」**：给编号操作步骤（还是不动蓝图）
- **「生成可执行步骤（DSL）」**：直接生成可执行的 DSL 并预览

## 4. 安全使用：出错怎么办

### 4.1 一键回滚

- 单步失败：直接看面板输出区的错误，决定修 prompt 还是跳过这一步。
- 批量出错后想回到干净状态：点 **「撤销本次 DSL 改动」** 按钮，会按事务数量连续 Ctrl+Z。

### 4.2 Ctrl+Z 始终可用

每一次"执行"都会被包成事务。标准的 `Ctrl+Z`、`Ctrl+Y` 都正常工作。

### 4.3 高风险步骤

某些步骤（如 `DestroyActor` / `OpenLevel` / `QuitGame` / `SetTimeDilation`）会在预览里被标为"高风险"。建议：

- **取消勾选**这些步骤，避免批量误触；
- 如必须执行，**请使用行末的「执行」按钮单步执行**，执行前确认当前蓝图是对的。

## 5. 反馈问题（强烈推荐）

发现问题时，不需要截图也不需要写长文：

1. 在面板上点 **「反馈本次问题」**。
2. 会自动生成一个 markdown 文件，路径类似：
  ```
   <Project>/Saved/BlueprintAIAssistant/feedback/feedback-YYYYMMDD-HHMMSS.md
  ```
3. 自动打开它所在的文件夹。
4. 用记事本或 VSCode 打开这个 md，在末尾的「补充说明（地编手动填写）」下面写 1-3 句话说明**你预期看到什么 / 实际遇到什么**。
5. 把这个 md 文件直接发到群里 / 邮件给开发同学。

文件里已经自动包含：当前 Provider、模型、prompt、DSL 步骤、最近面板输出/错误 —— 开发同学可以直接定位。

## 6. 查看使用日志（供好奇/排查用）

面板底部 **「打开日志目录」** 按钮，会弹出：

```
<Project>/Saved/BlueprintAIAssistant/
├── usage-20260407.log           # 今日使用日志（JSON Lines）
└── feedback/                    # 反馈 md 存放处
```

`usage-*.log` 里一行一条 JSON，包含：你点了哪个场景按钮、模型请求是否成功、DSL 解析步数、批量/单步执行是否成功等。**地编不需要看这里**，但开发会根据这些日志迭代工具。

## 7. 常见问题 FAQ

**Q：点"生成可执行步骤（DSL）"后面板一直显示"请求中"**
A：等待模型响应。国内模型一般 3-20 秒；如果 30 秒没反应，可能网络/余额问题，可以先点一次「连接测试」。

**Q：DSL 解析失败怎么办？**
A：面板会显示模型原始输出。大部分情况是模型"一不小心加了 Markdown 代码块标记"，重新点一次生成按钮即可。

**Q：某个步骤报 "未找到 fromNodeId=XXX 对应节点"**
A：通常是模型用了本来不存在的 id。你可以：

- 从该步骤开始单步执行前几步，观察节点有没有被创建；
- 或者修一下 prompt 明确说"用 BeginPlay 作为起点"再重新生成。

**Q：SpawnActor 提示"不在白名单"**
A：目前 `SpawnActorFromClass` 仅允许 `BP_Cursor` / `BP_Door_01` / `BP_FireBall`。其他类需要由开发同学扩白名单（出于安全考虑），请反馈具体要加哪个类。

**Q：我点了"撤销本次 DSL 改动"之后少撤销了**
A：如果你在执行 DSL 过程中手动做过其他编辑（移动节点、编辑变量等），它们都算事务。按钮只撤销和 DSL 关联的事务数量，不够的话继续按 `Ctrl+Z` 即可。

**Q：我想改内置的快捷按钮场景**
A：目前是硬编码的（`BlueprintAISceneTemplates.cpp`），告诉开发同学你要加/改哪个场景即可。
